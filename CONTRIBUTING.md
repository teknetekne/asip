ASIP is built on solidarity, not profit. We welcome contributions that align with our values:

- **Decentralization**: Keep it P2P
- **Security**: No shortcuts
- **Accessibility**: Simple for newcomers
- **Solidarity**: Help each other

## Code of Conduct

1. **Respect**: Treat everyone with respect
2. **Collaboration**: We build together
3. **Quality**: Write clean, tested code
4. **Documentation**: Explain your work

## How to Contribute

### Reporting Bugs

Open an issue with:
- Clear description
- Steps to reproduce
- Expected vs actual behavior
- Your environment (OS, Node version)

### Suggesting Features

Before suggesting:
1. Check existing issues
2. Align with ASIP principles
3. Explain the use case

### Pull Requests

1. Fork the repo
2. Create a feature branch
3. Write tests
4. Update docs
5. Submit PR

**PR Checklist:**
- [ ] Code follows our style
- [ ] Tests pass
- [ ] Documentation updated
- [ ] Commit messages clear

## Development Setup

```bash
git clone https://github.com/emektekneci/agent-international.git
cd agent-international
npm install
npm test
```

## Testing

```bash
# Run all tests
npm test

# Run with coverage
npm run test:coverage

# Run specific test
npm test -- network.test.js
```

## Code Style

- **ES6+**: Use modern JavaScript
- **Comments**: JSDoc for public APIs
- **Naming**: Descriptive, camelCase
- **Modules**: Single responsibility

## Commit Messages

Format: `<type>: <description>`

Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `test`: Tests
- `refactor`: Code restructure
- `chore`: Maintenance

Example:
```
feat: add rate limiting to security layer
```
